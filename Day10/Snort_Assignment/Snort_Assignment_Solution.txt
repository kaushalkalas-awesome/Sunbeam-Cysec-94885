Day10 - Snort Assignment

alert tcp any any -> $HOME_NET any (msg:"TCP Port Scan Detected"; flags:S; 
detection_filter: track by_src, count 10, seconds 5; sid:1000002; rev:1;)

1. alert tcp
Ans: This creates an alert for TCP traffic.

2. any any -> $HOME_NET any
Ans: This means traffic from any IP and any port going to your home network on any port.

3. msg:"TCP Port Scan Detected"
Ans: This is the message shown when the alert triggers.

4. flags:S;
Ans: This checks for TCP SYN packets, which start TCP connections.

5. detection_filter:track by_src, count 10, seconds 5;
Ans: This triggers an alert when the same source IP sends 10 SYN packets within 5 seconds.

6. sid:1000002;
Ans: This is the unique identifier for the rule.

7. rev:1;
Ans: This shows the version of the rule.

Q. What is IDS?
Ans: An IDS (Intrusion Detection System) monitors network traffic and alerts when suspicious activity is detected.

Q. Difference between IDS and IPS
Ans: IDS only detects and alerts.
IPS detects and actively blocks malicious traffic.

Q. What is Snort?
Ans: Snort is an open-source IDS/IPS used to analyze network traffic in real time.

Q. What is port scanning?
Ans: Port scanning is a method used to find open ports on a system.

Q. What is Nmap used for?
Ans: Nmap is used for network discovery and port scanning.

Q. What is HOME_NET?
Ans: HOME_NET is a Snort variable that defines the internal network you want to protect.

Q. What is a Snort rule?
Ans: A Snort rule is a set of conditions that tells Snort what traffic to detect or alert on.

Q. What is SID in Snort?
Ans: SID (Snort ID) is a unique number that identifies each Snort rule.
